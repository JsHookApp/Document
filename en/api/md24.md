# r2 — r2frida Reverse Engineering Engine

Built on the [r2frida](https://github.com/nowsecure/r2frida) project, this API provides low-level reverse engineering capabilities such as disassembly, memory search, module analysis, function tracing and interception.
All methods run directly in the Frida GumJS runtime — no Java-layer bridging is required.

---

## Disassembly

### `r2.disasm(address, count)`

Disassembles the machine-code instructions at the given address.

| Parameter | Type | Description |
|-----------|------|-------------|
| address | `string` / `NativePointer` | Start address |
| count | `number` | Number of instructions (default 32) |

**Returns**: `string` — Disassembly text, including addresses, mnemonics, operands and symbol information.

```javascript
var base = Module.findBaseAddress("libil2cpp.so");
var asm = r2.disasm(base.add(0x1234), 10);
console.log(asm);
// ;;; libil2cpp.so
// 0x71a2b234  mov  x0, x1
// 0x71a2b238  bl   #0x71a2c000
// ...
```

## Memory

### `r2.hexdump(address, size)`

Dumps memory contents in hexadecimal.

| Parameter | Type | Description |
|-----------|------|-------------|
| address | `string` / `NativePointer` | Start address |
| size | `number` | Number of bytes (default 256) |

```javascript
r2.hexdump(base.add(0x1000), 64);
```

### `r2.memoryRanges()`

Lists every memory region in the process.

**Returns**: `string` — One region per line, formatted as `base - end protection [path]`.

### `r2.memoryRangesJson()`

Returns the list of memory regions in JSON format.

### `r2.memoryRangesHere(address)`

Finds the memory region that contains the given address.

### `r2.memoryMaps()`

Lists the memory maps, merging adjacent regions that share the same path.

### `r2.allocate(size)`

Allocates a block of memory of the given size on the heap.

| Parameter | Type | Description |
|-----------|------|-------------|
| size | `number` | Number of bytes |

**Returns**: `string` — The address of the allocated block.

### `r2.allocateString(str)`

Allocates a UTF-8 string on the heap.

**Returns**: `string` — The address of the string.

### `r2.freeAlloc(addr)`

Frees memory that was previously allocated via `allocate` / `allocateString`.

### `r2.listAllocs()`

Lists every memory address allocated through r2.

### `r2.changeProtection(addr, size, perm)`

Changes the protection attributes of a memory region.

| Parameter | Type | Description |
|-----------|------|-------------|
| addr | `string` | Memory address |
| size | `number` | Size |
| perm | `string` | Permission string, e.g. `"rwx"`, `"r-x"` |

## Search

### `r2.searchString(pattern)`

Searches readable memory regions for a string.

| Parameter | Type | Description |
|-----------|------|-------------|
| pattern | `string` | The string to search for |

**Returns**: `string` (JSON) — A list of hits in the form `[{address, size}]`.

```javascript
var hits = JSON.parse(r2.searchString("GameManager"));
hits.forEach(function(h) {
    console.log("Found at: " + h.address);
});
```

### `r2.searchHex(hex)`

Searches for a hexadecimal pattern.

```javascript
var hits = JSON.parse(r2.searchHex("48 89 5c 24 08"));
```

### `r2.searchValue4(value)`

Searches for a 4-byte (32-bit, little-endian) value.

```javascript
// Search for the coin value 999
var hits = JSON.parse(r2.searchValue4(999));
```

### `r2.searchValue8(value)`

Searches for an 8-byte (64-bit, little-endian) value.

### `r2.searchWide(pattern)`

Searches for a UTF-16 wide string.

### `r2.searchStrings(minLen)`

Scans memory for all printable strings.

| Parameter | Type | Description |
|-----------|------|-------------|
| minLen | `number` | Minimum length (default 10) |

**Returns**: `string` (JSON) — `[{address, text}]`

## Modules / Symbols

### `r2.modules()`

Lists every loaded module.

**Returns**: `string` — One module per line: `base size name path`.

### `r2.modulesJson()`

Returns the module list in JSON format.

### `r2.symbols(moduleName)`

Lists the symbols of the given module.

```javascript
r2.symbols("libil2cpp.so");
```

### `r2.exports(moduleName)`

Lists the exported functions of the given module.

```javascript
r2.exports("libc.so");
```

### `r2.imports(moduleName)`

Lists the imported functions of the given module.

### `r2.lookupAddress(address)`

Looks up the symbol name and module that correspond to an address.

```javascript
var info = r2.lookupAddress(base.add(0x5678));
console.log(info); // "libil2cpp.so il2cpp_init 0x71a2b5678"
```

### `r2.lookupExport(name, moduleName)`

Looks up the address of an exported function. `moduleName` is optional; if omitted, all modules are searched.

```javascript
var addr = r2.lookupExport("il2cpp_string_new", "libil2cpp.so");
```

### `r2.lookupSymbol(name, moduleName)`

Looks up the address of a symbol.

## Class Enumeration (Java)

### `r2.classes()`

Lists every loaded Java class.

### `r2.classMethods(className)`

Lists all methods of the given Java class.

```javascript
var methods = r2.classMethods("com.example.GameActivity");
console.log(methods);
```

### `r2.classFields(className)`

Lists all fields of the given Java class.

### `r2.classLoaders()`

Lists every instantiated ClassLoader.

## Tracing

### `r2.trace(address)`

Traces calls to a function, automatically printing the call stack, arguments and return value.

Prefix the target with `java:` to trace a Java method.

```javascript
// Trace a native function
r2.trace(base.add(0x1234));

// Trace a Java method
r2.trace("java:com.example.App.checkLicense");
```

### `r2.listTraces()`

Lists all currently active trace points.

### `r2.clearTrace(address)`

Clears the trace at the given address.

### `r2.clearAllTraces()`

Clears every trace.

---

## Interception

### `r2.interceptRet0(target)`

Intercepts a function and forces it to return 0. Supports the `java:` prefix.

```javascript
r2.interceptRet0(base.add(0x5678));
r2.interceptRet0("java:com.example.App.isVip");
```

### `r2.interceptRet1(target)`

Intercepts a function and forces it to return 1.

### `r2.interceptRetInt(target, value)`

Intercepts a function and forces it to return the given integer value.

### `r2.interceptRetString(target, str)`

Intercepts a function and forces it to return the given string.

### `r2.interceptDetachAll()`

Removes every interception.

## Stalker

### `r2.stalkFunction(address, timeout)`

Performs instruction-level function tracing using Frida's Stalker (experimental).

| Parameter | Type | Description |
|-----------|------|-------------|
| address | `string` | Function address |
| timeout | `number` | Timeout in seconds (default 10) |

**Returns**: `Promise<string>` — A call summary (count × symbol name).

```javascript
r2.stalkFunction(base.add(0x1234), 5).then(function(result) {
    console.log(result);
});
```

## General

### `r2.info()`

Returns a JSON string with process information (PID, architecture, platform, etc.).

### `r2.version()`

Returns the r2frida version number.

### `r2.cmd(command)`

Runs an r2-style command string. Aimed at advanced users already familiar with radare2.

```javascript
r2.cmd("pd 0x71a2b234 20");    // disassemble
r2.cmd("il");                   // module list
r2.cmd("/x 48 89 5c 24 08");   // hex search
r2.cmd("dm");                   // memory maps
r2.cmd("dt 0x71a2b234");       // trace
r2.cmd("di0 0x71a2b234");      // intercept, return 0
r2.cmd("ic");                   // Java class list
```

Supported commands: `pd`, `px`, `dm`, `dmj`, `dm.`, `dmm`, `dma`, `dmas`, `dma-`, `dmal`, `dmp`, `/`, `/x`, `/v4`, `/v8`, `/w`, `/z`, `il`, `ilj`, `is`, `iE`, `ii`, `fd`, `iaE`, `isa`, `ic`, `icm`, `icf`, `icL`, `dt`, `dt-`, `dt-*`, `di0`, `di1`, `dii`, `dis`, `di-1`, `di-*`, `dr`, `dpt`, `dptj`, `ls`, `cat`, `dd`, `env`, `dl`, `dp`, `i`, `?V`

## Complete Examples

### Example 1: Analyzing a native library

```javascript
// Find the module base address
var base = Module.findBaseAddress("libgame.so");

// Disassemble the entry point
console.log(r2.disasm(base, 20));

// List the exports
console.log(r2.exports("libgame.so"));

// Search for a string
var hits = JSON.parse(r2.searchString("LevelManager"));
console.log("Found " + hits.length + " hits");
```

### Example 2: Searching for and modifying a memory value

```javascript
// Search for the coin value
var hits = JSON.parse(r2.searchValue4(9999));
console.log("Found " + hits.length + " addresses");

// Inspect each hit with a hexdump
hits.forEach(function(h) {
    console.log(r2.hexdump(h.address, 32));
});
```

### Example 3: Analyzing a Java method

```javascript
// List the methods of a class
console.log(r2.classMethods("com.example.GameActivity"));

// Trace a method
r2.trace("java:com.example.GameActivity.getScore");

// Intercept a method's return value
r2.interceptRet1("java:com.example.GameActivity.isPremium");
```

### Example 4: Command mode

```javascript
// Use the r2 command style
console.log(r2.cmd("il"));          // module list
console.log(r2.cmd("is libc.so"));  // libc symbols
console.log(r2.cmd("dm"));          // memory maps
console.log(r2.cmd("dpt"));         // thread list
console.log(r2.cmd("ic"));          // Java class list
```
